#1. Two Sum

Easy Arrays Hashing
⏱ 5 min read 📐 3 approaches 🏢 Google · Amazon · Meta ✅ ~49% acceptance
Try on LeetCode ↗

Problem Statement

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice. You can return the answer in any order.

Example 1:
Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: Because nums[0] + nums[1] == 9, we return [0, 1].

Example 2:
Input: nums = [3,2,4], target = 6
Output: [1,2]

Constraints: 2 ≤ nums.length ≤ 10⁴ · -10⁹ ≤ nums[i] ≤ 10⁹ · -10⁹ ≤ target ≤ 10⁹ · Exactly one valid answer exists.

Intuition — Building the Insight

Before jumping to code, let's build the core intuition. This is the step most people skip, and it's exactly why they struggle to solve new problems.

Read the problem again: we want two numbers in the array that sum to a target. The naive question to ask: "can I just check every pair?"

Yes — but that's O(n²). With n up to 10,000, that's up to 100 million operations. LeetCode will pass it, but interviewers expect better.

Core Insight

For every number x we encounter, we don't need to search the whole array for its complement — we just need to know if target - x has appeared before. A hash map tells us this in O(1).

This reframe — from "find a pair" to "check if complement exists in memory" — is the entire algorithm. Everything else is mechanics.

Approach 1 — Brute Force

For each element nums[i], scan every element nums[j] where j > i. Check if they sum to target.

Why it works: You check every possible pair, so you'll definitely find the one that sums to target.
Why it's slow: n elements × n elements = O(n²) checks.

Python 
def twoSum(nums, target):
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []  # guaranteed to find answer per constraints

Complexity: Time O(n²) · Space O(1). Passes LeetCode but will fail in interviews for large inputs.

Approach 2 — Sort + Two Pointers

Sort the array. Then use two pointers: one at the start (left) and one at the end (right). Move them based on whether the current sum is too small or too large.

The catch: Sorting changes the original indices. Since the problem asks for indices, you need to store index-value pairs before sorting — this means O(n) extra space.

Python 
def twoSum(nums, target):
    # Store (value, original_index) pairs
    indexed = sorted(enumerate(nums), key=lambda x: x[1])

    left, right = 0, len(indexed) - 1

    while left < right:
        current_sum = indexed[left][1] + indexed[right][1]

        if current_sum == target:
            return [indexed[left][0], indexed[right][0]]
        elif current_sum < target:
            left += 1   # need bigger sum
        else:
            right -= 1  # need smaller sum

    return []

Complexity: Time O(n log n) · Space O(n). Better than brute force, but not optimal. This pattern is extremely useful for variants where you don't need original indices (e.g., 3Sum, 4Sum).

Approach 3 — HashMap (One Pass) ★ Optimal

Iterate through the array once. For each element nums[i], calculate diff = target - nums[i]. If diff is already in our hash map, we found the pair. Otherwise, add nums[i] to the map (mapping value to index) and continue.

Why this works

We process elements left to right. When we're at index i, every element with index j < i is already in the map. So when we look up diff, we're guaranteed it came before i — no duplicate index possible.

Python 
def twoSum(nums, target):
    seen = {}  # value : index

    for i, n in enumerate(nums):
        diff = target - n

        if diff in seen:
            return [seen[diff], i]  # found it!

        seen[n] = i  # remember this value for later

    return []  # unreachable per problem constraints
Java 
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>();

    for (int i = 0; i < nums.length; i++) {
        int diff = target - nums[i];

        if (seen.containsKey(diff)) {
            return new int[]{seen.get(diff), i};
        }

        seen.put(nums[i], i);
    }

    return new int[]{};
}
C++ 
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> seen;

    for (int i = 0; i < nums.size(); i++) {
        int diff = target - nums[i];

        if (seen.count(diff)) {
            return {seen[diff], i};
        }

        seen[nums[i]] = i;
    }

    return {};
}

Complexity: Time O(n) · Space O(n). Optimal for this problem.

Edge Cases to Test

Never submit without running through these mentally:

1. Duplicate values: nums = [3, 3], target = 6 → Should return [0, 1]. The one-pass solution handles this because when we see the second 3, the first 3 is already in seen.

2. Negative numbers: nums = [-1, -2, -3, -4, -5], target = -8 → Works perfectly, hash maps handle negative keys fine.

3. Answer at the very end: nums = [1, 2, 3, 4, 5], target = 9 → Returns [3, 4]. The algorithm doesn't shortcut early — it scans through all elements if needed.

4. Can I use the same element twice? The problem explicitly says no. The one-pass approach is safe: seen[n] = i is only added after checking for the complement, so we never use index i for both values.

Interview Follow-Up Questions

Interviewers rarely stop at the basic solution. Expect at least one of these:

  • ❓
    What if there are multiple valid pairs? Return all pairs. Change return to append to a list and continue iterating.
  • ❓
    What if no solution exists? Return [-1, -1] or an empty array. The problem guarantees exactly one, but always clarify this assumption.
  • ❓
    3Sum — what if we need 3 numbers? Fix one number, then Two Sum on the rest. Use sort + two pointers. See problem #15.
  • ❓
    What if the array is huge and you can only read it once? The one-pass HashMap approach already handles this — it's a streaming algorithm.
  • ❓
    Can you do it in O(1) space? Only if you can sort and modify the array in place (Two Pointers), and you don't need original indices.

The Pattern — Where Else Does This Apply?

Pattern Recognition

The "complement in a hash map" pattern generalizes to any problem where you need to find pairs/triples with a property. Two Sum is the entry point. Master the intuition here, and 3Sum, 4Sum, Two Sum II, and Subarray Sum Equals K all become easier.
