#121. Best Time to Buy and Sell Stock

Easy Arrays Sliding Window Greedy
⏱ 5 min read 📐 2 approaches 🏢 Amazon · Goldman · Facebook · Apple ✅ ~54% acceptance
Try on LeetCode ↗

Problem Statement

You are given an array prices where prices[i] is the price of a given stock on the ith day.

You want to maximize your profit by choosing a single day to buy one stock and choosing a different day in the future to sell that stock.

Return the maximum profit you can achieve. If you cannot achieve any profit, return 0.

Example 1:
Input: prices = [7,1,5,3,6,4]
Output: 5
Buy on day 2 (price = 1), sell on day 5 (price = 6). Profit = 6 − 1 = 5.

Example 2:
Input: prices = [7,6,4,3,1]
Output: 0
Prices only decrease — no profitable transaction is possible.

Constraints: 1 ≤ prices.length ≤ 10⁵ · 0 ≤ prices[i] ≤ 10⁴

Intuition — Think Like a Trader

Let's reason about this the way a human would. As you read prices from left to right (day by day), you're looking for the biggest gap between a past low and a future high.

Here's the key observation: if you're on day i and thinking about selling, the best you could have done is to have bought at the lowest price seen so far (days 0 through i−1). That's it. That's the entire algorithm.

The One Insight

As you scan prices left to right, maintain min_price — the cheapest buy opportunity you've seen so far. At each day, the best profit if you sold today is prices[i] − min_price. Track the global maximum of this across all days. Update min_price if today's price is lower.

Notice: you never have to check all pairs. You're making a greedy decision: "the best buy price for any future sale is the minimum price seen before that sale day." One scan captures all possibilities.

This problem is a cousin of Kadane's Algorithm — if you convert the prices array to a daily profit array (difference between consecutive days), the maximum subarray sum equals the maximum buy-sell profit. Understanding this connection opens up the entire "stock problems" family.

Approach 1 — Brute Force

Try every pair (buy day i, sell day j where j > i). Track the maximum price[j] − price[i].

Python
def maxProfit(prices):
    max_profit = 0
    n = len(prices)
    for i in range(n):
        for j in range(i + 1, n):
            profit = prices[j] - prices[i]
            max_profit = max(max_profit, profit)
    return max_profit

Complexity: Time O(n²) · Space O(1). Times out for large inputs.

Approach 2 — One Pass, Track Minimum ★

Python
def maxProfit(prices):
    min_price = float('inf')  # cheapest buy seen so far
    max_profit = 0

    for price in prices:
        if price < min_price:
            min_price = price          # found a cheaper buy day
        elif price - min_price > max_profit:
            max_profit = price - min_price  # better profit if sold today

    return max_profit
Java
public int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE;
    int maxProfit = 0;

    for (int price : prices) {
        if (price < minPrice) {
            minPrice = price;
        } else if (price - minPrice > maxProfit) {
            maxProfit = price - minPrice;
        }
    }
    return maxProfit;
}
C++
int maxProfit(vector<int>& prices) {
    int minPrice = INT_MAX, maxProfit = 0;
    for (int p : prices) {
        minPrice = min(minPrice, p);
        maxProfit = max(maxProfit, p - minPrice);
    }
    return maxProfit;
}

Complexity: Time O(n) · Space O(1). One pass, two variables.

Edge Cases to Test

1. All decreasing prices: [7,6,5,4,3] → 0. Every day you compute a negative profit — max_profit stays 0. This is why we initialize max_profit = 0, not −∞.

2. Single element: [5] → 0. You need at least one buy and one sell day.

3. Two elements — profit possible: [1, 5] → 4. Works: min_price=1 on day 0, profit=4 on day 1.

4. All same price: [3,3,3,3] → 0. No profit, returns 0 correctly.

  • ❓

  • ❓

  • ❓
    How is this related to Kadane's Algorithm? Convert prices to daily differences: diff[i] = prices[i] - prices[i-1]. The maximum buy-sell profit = maximum subarray sum of diff. Kadane's Algorithm solves that in O(n). They're the same problem in disguise.

The Pattern — Track Running Optimum

Pattern: Running Min/Max + Greedy Choice

When you need the best outcome from a sequence where each step depends on a previously seen "optimal value" (like the cheapest buy price), scan once and maintain that running optimal. This powers: Jump Game (max reachable index), Gas Station (running prefix sum with tracking), and Trapping Rain Water. The unifying idea: you don't need to look backwards if you've already stored the best backwards value.
