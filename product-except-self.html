#238. Product of Array Except Self

Medium Arrays Prefix Product
⏱ 7 min read 📐 3 approaches 🏢 Amazon · Meta · Microsoft · Apple ✅ ~65% acceptance
Try on LeetCode ↗

Problem Statement

Given an integer array nums, return an array answer such that answer[i] is equal to the product of all elements of nums except nums[i].

The product of any prefix or suffix of nums is guaranteed to fit in a 32-bit integer. You must write an algorithm that runs in O(n) time and without using the division operation.

Example 1:
Input: nums = [1,2,3,4]
Output: [24,12,8,6]
Explanation: answer[0] = 2×3×4 = 24, answer[1] = 1×3×4 = 12, etc.

Example 2:
Input: nums = [-1,1,0,-3,3]
Output: [0,0,9,0,0]

Constraints: 2 ≤ nums.length ≤ 10⁵ · −30 ≤ nums[i] ≤ 30 · No division operation allowed.

Intuition — The Way Andrew Ng Would Teach It

Let's first ask: what do you naturally want to do? Compute the total product, then divide by each element. That's the obvious idea. But:

  1. What if an element is zero? Division by zero breaks everything.
  2. The problem explicitly forbids division.

So we need a different mental model. Here's the key question: for position i, what exactly is answer[i]?

The Core Insight

answer[i] = (product of everything to the LEFT of i) × (product of everything to the RIGHT of i).

These are the prefix product and suffix product at position i. Compute both separately — then multiply them together. No division needed, ever.

Think of it visually. For nums = [1, 2, 3, 4] and position i = 2 (value 3):

  • Left of i=2: 1 × 2 = 2
  • Right of i=2: 4 = 4
  • Answer[2] = 2 × 4 = 8 ✓

The O(1) space trick: instead of storing full prefix and suffix arrays, do two passes. First pass L→R to build prefix products into the output array. Second pass R→L multiplies in the suffix product using a running variable. Beautiful — you only need the output array.

Approach 1 — Brute Force

For each index i, loop through all other elements and multiply. O(n²) — shows you understand the problem but not the optimal structure.

Python
def productExceptSelf(nums):
    n = len(nums)
    answer = [1] * n
    for i in range(n):
        for j in range(n):
            if i != j:
                answer[i] *= nums[j]
    return answer

Complexity: Time O(n²) · Space O(1) extra (O(n) for output).

Approach 2 — Prefix + Suffix Arrays

Build a prefix array where prefix[i] = product of all elements to the left of i. Build a suffix array where suffix[i] = product of all elements to the right of i. Answer[i] = prefix[i] × suffix[i].

Python
def productExceptSelf(nums):
    n = len(nums)
    prefix = [1] * n   # prefix[i] = product of nums[0..i-1]
    suffix = [1] * n   # suffix[i] = product of nums[i+1..n-1]

    for i in range(1, n):
        prefix[i] = prefix[i-1] * nums[i-1]

    for i in range(n-2, -1, -1):
        suffix[i] = suffix[i+1] * nums[i+1]

    return [prefix[i] * suffix[i] for i in range(n)]

Complexity: Time O(n) · Space O(n). Good stepping stone for understanding — mention it before landing on the optimal.

Approach 3 — Two-Pass, O(1) Extra Space ★

Pass 1 (left → right): Fill output[i] with the product of all elements to the LEFT of i. Initially output[0] = 1 (nothing to its left).

Pass 2 (right → left): Maintain a running suffix product. Multiply it into output[i], then update it by multiplying nums[i].

Python
def productExceptSelf(nums):
    n = len(nums)
    output = [1] * n

    # Pass 1: fill output[i] with product of everything LEFT of i
    prefix = 1
    for i in range(n):
        output[i] = prefix
        prefix *= nums[i]

    # Pass 2: multiply in product of everything RIGHT of i
    suffix = 1
    for i in range(n - 1, -1, -1):
        output[i] *= suffix
        suffix *= nums[i]

    return output
Java
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] output = new int[n];

    int prefix = 1;
    for (int i = 0; i < n; i++) {
        output[i] = prefix;
        prefix *= nums[i];
    }

    int suffix = 1;
    for (int i = n - 1; i >= 0; i--) {
        output[i] *= suffix;
        suffix *= nums[i];
    }

    return output;
}

Complexity: Time O(n) · Space O(1) extra (the output array is required by the problem). Two passes. Industry-clean solution.

Edge Cases to Test

1. Array with zeros: [1, 0, 3, 4] → [0, 12, 0, 0]. The prefix×suffix approach handles zeros naturally — no division-by-zero risk. This is exactly why division fails but prefix products don't.

2. Two zeros: [0, 0, 3, 4] → [0, 0, 0, 0]. If there are two zeros, every product is 0 (always some zero in the product). The algorithm handles this automatically.

3. Negative numbers: [-1, 1, 2, 3] → [6, -6, -3, -2]. Signs work perfectly — just multiplication.

4. Length 2: [3, 4] → [4, 3]. The prefix of index 0 = 1, suffix of index 1 = 1. Works correctly.

Interview Follow-Up Questions

  • ❓
    What if division was allowed? Compute total product. For each index, answer[i] = total / nums[i]. Handle zeros separately: if there's exactly one zero, everything else is 0, and the zero-index position gets the total product of non-zeros. If two zeros, everything is 0.
  • ❓
    Can you do this in one pass? Not really — you need information about both the left AND right of each element, and the right information isn't available until you've scanned right. Two passes is the minimum.
  • ❓
    What if you needed the sum (not product) of all except self? Total sum minus nums[i]. O(n), trivial — but good thinking!
  • ❓
    What if the array doesn't fit in memory? Stream in two oppositely-directed passes. Store prefix products in a pass, then compute suffix on the fly in the second pass — exactly what this optimal solution already does.

The Pattern — Prefix + Suffix Decomposition

Pattern: Left-Right Decomposition

When the answer at position i depends on everything to its left AND everything to its right, the pattern is: precompute left contributions in a forward pass, right contributions in a backward pass. This technique generalizes to: Trapping Rain Water (max from left, max from right), Best Time to Buy & Sell Stock (min from left, max from right in some formulations), and Candy Distribution (left pass + right pass).
