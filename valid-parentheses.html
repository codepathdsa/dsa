#20. Valid Parentheses

Easy Stack String
⏱ 5 min read 📐 1 optimal approach 🏢 Google · Amazon · Meta · Microsoft ✅ ~40% acceptance
Try on LeetCode ↗

Problem Statement

Given a string s containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid.

An input string is valid if:

  1. Open brackets must be closed by the same type of brackets.
  2. Open brackets must be closed in the correct order.
  3. Every close bracket has a corresponding open bracket of the same type.

Example 1: s = "()" → true

Example 2: s = "()[]{}" → true

Example 3: s = "(]" → false

Constraints: 1 ≤ s.length ≤ 10⁴ · s contains only ()[] characters.

Intuition — The Stack as a "Memory of Open Brackets"

This problem is simpler than it looks once you find the right mental model. Ask yourself: when you encounter a closing bracket like ), what's the only acceptable open bracket it can close?

It must close the most recently opened bracket. That's the key. Order matters — ([)] is invalid because the most recently opened bracket is [, but ) is trying to close it.

The Stack as a Mind Map

A stack is exactly "the most recently opened bracket, at the top." When you see an open bracket, push it — you're saying "I'm waiting for this to close later." When you see a close bracket, pop from the stack — "this must close the most recent open bracket." If the types don't match, invalid. If the stack is empty when you pop, invalid. If the stack is non-empty at the end, invalid.

This is one of the most important patterns in CS. Stacks are used everywhere when you need to match things in LIFO order: compilers (syntax validation), HTML parsers (tag matching), expression evaluators, undo/redo systems. Master this here.

Stack Approach — The Clear, Fast Solution

Use a hashmap to map each closing bracket to its corresponding opener. When you encounter a closing bracket, check: does the top of the stack hold the expected open bracket?

Python
def isValid(s):
    stack = []
    close_to_open = {
        ')': '(',
        ']': '[',
        '}': '{'
    }

    for char in s:
        if char in close_to_open:
            # It's a closing bracket — pop and validate
            if not stack or stack[-1] != close_to_open[char]:
                return False
            stack.pop()
        else:
            # It's an opening bracket — remember it
            stack.append(char)

    return len(stack) == 0  # valid iff all brackets were closed
Java
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    Map<Character, Character> map = Map.of(
        ')', '(', ']', '[', '}', '{'
    );
    for (char c : s.toCharArray()) {
        if (map.containsKey(c)) {
            if (stack.isEmpty() || stack.peek() != map.get(c))
                return false;
            stack.pop();
        } else {
            stack.push(c);
        }
    }
    return stack.isEmpty();
}
C++
bool isValid(string s) {
    stack<char> stk;
    unordered_map<char,char> m = {{')','('},{']','['},{'}','{'}};
    for (char c : s) {
        if (m.count(c)) {
            if (stk.empty() || stk.top() != m[c]) return false;
            stk.pop();
        } else stk.push(c);
    }
    return stk.empty();
}

Complexity: Time O(n) · Space O(n) for the stack. Each character is processed exactly once.

Edge Cases to Test

1. Close bracket with empty stack: "]" → false. Stack is empty when we try to pop — handled by the not stack check.

2. Odd length: Any string of odd length MUST be invalid — you can't pair all brackets. A quick early return: if len(s) % 2 != 0: return False. Optional optimization.

3. All opens, no closes: "(((" → false. Loop ends. Stack = ['(','(','(']. len(stack) != 0 → False. ✓

4. Nested, all valid: "{[()]}" → true. Perfect nesting, each bracket matches its corresponding opener.

The Pattern — Stack for Matching & Nesting

Pattern: Stack for LIFO Matching

Any time you have elements that must be closed in reverse order of opening, you need a stack. The stack holds "things I'm currently inside" and pops as you exit them. This pattern is used in: Decode String (#394: nested encoding), Largest Rectangle in Histogram (#84: monotonic stack), Daily Temperatures (#739: next greater element), and Evaluate Reverse Polish Notation (#150: operand/operator stack). Valid Parentheses is the simplest and most important entry point into this category.
